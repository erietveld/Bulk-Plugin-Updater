Initial test version for Internal ServiceNow release

The Repository is the ServiceNow IDE version of the application. So you need to link a fork of the repository to your personal ServiceNow IDE.


Installation instructions

In Github:
	• Fork this repository
  • Setup a user with a token so you can access the repository

In the ServiceNow IDE:
	• Create a new Workspace
	• It will open and on the right, you'll see 'Clone Git repository'
	• It will ask for the Repo URL, use the link to the Forked repo
  • If the credentials are set it will pull your application in. Otherwise, it will ask you for the credentials
	• Ask BA to Install dependencies
	• Do a Build and Deploy/Install

  Have fun!
